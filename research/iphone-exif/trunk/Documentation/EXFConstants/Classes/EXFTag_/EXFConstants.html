EXFTag

Definition data of an EXF Tag

Superclass: NSObject
Declared In: EXFConstants.h


The EXFTag consists of an tagId, dataType, shortName, parentTagId, whether it is user editable and the number of components that each tag consists of.

The dataType can only be one of the valid EXFDataType enum values. The shortName is as specified in the EXF specification. If localised or more user readable names are required you should use these as the key values to the localised form. The parentTagId shows the hierarchical parent Tag of ach EXFTag. This is required in order to work out which directory or subdirectory a tag value should be inserted into. Editable tags are those that can be altered or add by users of the library. Attempting to alter a non-writable tag will result in an exception. Components defines the number of instances of each data type. As each data type is a certain number of bytes the actual byte size occupied for each tag is components * dataType size. For more detail see the EXF specification http://www.exif.org/Exif2-2.PDF
